# Git
### Всё, что связано с гитом (`Git`)
</div>

>[!note]
>### **Определение**
> **Git (Global Information Tracker)** - это распределённая система контроля версий **(Version Control System, VCS)**. Она предназначена для отслеживания изменений в исходном коде и других файлах проекта в процессе разработки.

По сути, репозиторий - это "*крутая*" папка

## Ключевые концепции:
1. **Хранилище (Repository)** - папка проекта, где Git отслеживает изменения

2. **Коммит (Commit)** - сохранённая версия проекта с описанием изменений

3. **Ветка (Branch)** - независимая линия разработки

4. **Слияние (Merge)** - объединение изменений из разных веток

5. **Удалённый репозиторий (Remote)** - копия проекта на сервере (например, GitHub)

### Для того, чтобы начать работать с гитом, нужно:
1. [**Установить его с оффициального сайта**](https://git-scm.com/install/windows)
>[!warning]
>### Важно по поводу установки
> В установщике **все** галочки стоят по умолчанию, достаточно просто нажимать "**Далее**"
2. После того, как мы установили `Git`, нужно создать репозиторий в любом папке.
Для того чтобы в любом папке создать репозиторий, нужно:
Нажать ПКМ на папку и нажать на: `Open Git Bash here`, далее, у вас откроется терминал,а там уже надо ввести:
```bash
git init
```
3. Чтобы, добавлять файлы для отслеживания нужно ввести:
```bash
git add .
```
4. **Создание коммитов**
```bash
git commit -m "Добавил документ"
```
>[!warning]
> ### По поводу коммитов
> Нужно **всегда** в коммитах, писать то, что вы добавили, иначе при откате на предыдущую версию, вы будете очень долго разбираться в коммитах.

>[!note]
> ### Правильные названия для коммитов
> - "Добавил `[Объект]`
> - "Изменил `[Объект]`
> - "Удалил `[Обьект]`
> - "Что-то сделал с `[Обьектом]`
> ### Нельзя:
> - Рандомные символы и циферки
> "30984503948 Мурино лее"

5. Добавляем связь с удаленным **репозиторием**
```bash
git remote add origin https://github.com/username/repo.git
```
6. Отправляем изменения на сервер
```bash
git push origin main
```
7. По желанию, добавляем имя пользователя в `гит`
```bash
git config --global user.name "Admin"
git config --global user.email "admin1488@murino.com"
```
8. Проверяем состояние репозитория:
```bash
git status
```
9. Добавляем файлы в коммит
```bash
git add [file_name]
```
10. Если всё сделали, можно выходить из `гита`
```bash
exit
```

### Преимущеста `Гита`:
1. История всех изменений с возможностью отката

2. Параллельная работа нескольких разработчиков

4. Резервное копирование кода

4. Управление экспериментальными функциями через ветки

# Bash
### Всё, что связано с Башом (`Bash`)
</div>

>[!note]
>### Определения
> **Bash (Bourne Again Shell)** - это командная оболочка (shell) и язык сценариев для Unix-подобных операционных систем (Linux, macOS, WSL). Это интерфейс между пользователем и операционной системой, позволяющий выполнять команды, автоматизировать задачи и писать скрипты.

### Ключевые возможности:

**Интерактивная оболочка** - выполнение команд в терминале

**Скриптинг** - написание исполняемых файлов для автоматизации

**Управление процессами** - запуск, остановка, управление программами

**Работа с файлами** - создание, редактирование, поиск файлов

**Переменные и управление потоком** - как в языках программирования

### Время, для базовых команд:
1. Показать текущую директорию
```bash
pwd
```
2. список файлов и папок
```bash
ls
```
3. переход в папку `projects`(как пример)
```bash
cd ~/projects
```
4. Переход на уровень выше
```bash
cd ..
```
### 5. Работа с файлами
создать пустой файл
```bash
touch file.txt
```
6. создать папку
```bash
mkdir folder
```
7. копировать файл
```bash
cp file.txt copy.txt
```
8. переместить/переименовать
```bash
mv file.txt new_name.txt
```
9. удалить файл
```bash
rm file.txt
```
### 10. Просмотр содержимого
вывести весь файл
```bash
cat file.txt
```
> почему кот я хз

11. вывести первые 5 строк
```bash
head -n 5 file.txt
```
12. вывести последние 5 строк
```bash
tail -n 5 file.txt
```
>[!note]
> ### head и tail
> вместо *5* можно вписать любое число, это для примера
13. Просмотр с прокруткой
```bash
less file.txt
```
### 14. Поиск
поиск `"text"` в файле
```bash
grep "text" file.txt
``` 
15. найти все `.py` файлы
```bash
find . -name "*.py"
```

## Время для `Bash`-скрипта
```bash
#!/bin/bash
# Это комментарий - скрипт для создания проекта

echo "Создание нового проекта..."
read -p "Введите имя проекта: " project_name

mkdir "$project_name"
cd "$project_name" || exit

# Создание структуры
mkdir src tests docs
touch README.md main.py requirements.txt

echo "Проект '$project_name' создан!"
echo "Структура:"
tree
```
### Переменные и управление
```bash
#!/bin/bash
name="Иван"          # переменная
echo "Привет, $name!"  # использование переменной

# Условные операторы
if [ -f "file.txt" ]; then
    echo "Файл существует"
else
    echo "Файл не найден"
fi

# Циклы
for i in {1..5}; do
    echo "Итерация $i"
done

# Функции
greet() {
    echo "Привет, $1!"
}
greet "Мир"  # Привет, Мир!
```
### Другие полезные возможности
```bash
# Перенаправление ввода/вывода
ls > files.txt      # вывод в файл
cat < input.txt     # ввод из файла
ls | grep ".py"     # конвейер (pipe)

# Фоновые процессы
python server.py &  # запуск в фоне
jobs               # список фоновых задач

# Переменные окружения
echo $PATH         # пути поиска программ
export NAME="Иван" # установка переменной окружения
```
>[!note]
> ### `#!/bin/bash` - Shebang (шебанг)
> Эта строка называется **shebang (хеш-банг)** или **hashbang**. Она указывает системе, какая программа должна выполнять этот скрипт.

### Что это значит?
`#!` - магическая последовательность, которую читает система

`/bin/bash` — путь к интерпретатору, который выполнит скрипт
### Зачем это нужно?
1. **Определение интерпретатора**
```bash
#!/bin/bash        # Исполнять через Bash
#!/usr/bin/env python  # Исполнять через Python
#!/usr/bin/node    # Исполнять через Node.js
```
2. Позволяет запускать скрипт как программу

### Без шебанга:
```bash
bash script.sh      # нужно явно указывать интерпретатор
```
### с Шебенгом и правами на выполнение:
```bash
chmod +x script.sh  # даём права на выполнение
./script.sh         # запускаем как программу
```
### Примеры разных шебангов:
```bash
#!/bin/bash           # Классический Bash
#!/usr/bin/env bash   # Более переносимый способ (ищет bash в PATH)
#!/usr/bin/python3    # Python 3 скрипт
#!/bin/sh            # POSIX-совместимый shell (более ограниченный)
#!/usr/bin/php       # PHP скрипт
```

>[!note]
> ### Итог шебанга:
> это указание системе, что этот текстовый файл нужно выполнять через интерпретатор Bash, а не просто читать как текст. Это делает скрипт самодостаточной программой.
### Вывод: Зачем нужен Bash?
1. **Автоматизация рутинных задач** (резервное копирование, деплой)

2. **Быстрый доступ к системным утилитам**

3. **Управление серверами через SSH**

4. **Обработка текстовых данных** (логи, конфиги)

5. **Интеграция в CI/CD пайплайны**

